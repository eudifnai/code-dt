# CodeDT Electron 现场终验口令版 v0.1

## 用法

打开 CodeDT 的 Electron 窗口后，直接按下面顺序操作。

每一步都只有三件事：

- 点哪里
- 看什么
- 什么算通过

---

## 1. 打开设置

点哪里：

- 右上角设置按钮

看什么：

- 弹出模型设置面板
- 能看到 `DeepSeek` 和 `OpenAI 兼容接口`

通过标准：

- 设置面板正常打开

---

## 2. 保存模型配置

点哪里：

- 选择一个 Provider
- 填好必要配置
- 点 `保存设置`

看什么：

- 有保存成功反馈
- Quickstart 或主界面不再像“卡在未配置模型”

通过标准：

- 设置能保存，没有阻塞错误

---

## 3. 打开工作区

点哪里：

- 左侧 `打开工作区`

看什么：

- 左侧出现真实项目名
- 文件树出现
- Git 区块开始有内容

通过标准：

- 工作区成功打开

---

## 4. 选择一个文本文件

点哪里：

- 左侧文件树里点一个小一点的文本文件

看什么：

- 右侧文件预览出现真实内容
- 上下文相关动作变得可用

通过标准：

- 文件预览正常显示

---

## 5. 把文件加入上下文并发一条消息

点哪里：

- 点 `加入上下文`
- 在输入框输入一句简单请求
- 点 `发送`

建议请求：

- `解释这个文件是做什么的`

看什么：

- 中间对话区出现模型回复
- 回复明显围绕当前文件

通过标准：

- 聊天链路正常

---

## 6. 生成 AI Draft

点哪里：

- 保持当前文件选中
- 输入一个小改动请求
- 触发 `AI Draft`

建议请求：

- `帮我做一个小的命名优化，并保持行为不变`

看什么：

- 右侧出现 draft
- 能看到 diff 或明显的改动内容

通过标准：

- Draft 生成成功，而且是先审查再落盘

---

## 7. 应用 Draft

点哪里：

- 点 `Apply Draft`

看什么：

- 文件预览刷新
- Git 状态刷新
- 右侧或附近出现验证接力提示

通过标准：

- 改动真正写回，并且界面状态跟着更新

---

## 8. 跑一条验证命令

点哪里：

- 在 `工作区命令` 区点击推荐命令或直接点 `运行`

看什么：

- 终端输出开始滚动
- Recent Runs 更新

通过标准：

- 命令能跑，结果能记录

---

## 9. 如果是前端项目，验 Preview

点哪里：

- `启动开发服务`
- `打开`
- `截图`

看什么：

- 右侧能看到页面
- 截图能生成

通过标准：

- Preview 和 Capture 链路可用

---

## 10. 验 Git 最小闭环

点哪里：

- `暂存`
- `取消暂存`
- 再次暂存
- 查看 commit draft
- 如果当前仓库适合测试，再点 `提交`

看什么：

- Git 计数变化合理
- commit 草稿可读
- 提交成功后最新提交信息刷新

通过标准：

- Git 基础工作流正常

---

## 11. 复制交付摘要

点哪里：

- `Handoff summary` 里的 `Copy`

看什么：

- 能复制成功
- 摘要里至少有 branch、验证上下文、最新提交上下文

通过标准：

- 交付摘要可复制、可读

---

## 最终判断

直接通过：

- 第 1 到 8 步通过
- 如果是前端项目，第 9 步通过
- 第 10 和 11 步能正常完成

可判定为部分通过：

- 工作区、聊天、draft、apply、验证链路都通过
- 但 Preview 或 Git 发布链路在当前测试项目上不适合完整验证

不通过：

- 工作区打不开
- 文件预览不正常
- Draft 不能生成或不能 Apply
- 验证命令完全跑不通
